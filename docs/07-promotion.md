# 07 — 推广计划

## 推广总纲
> 用免费工具获取流量 → 用优质内容建立信任 → 转化为付费用户

## 一、搜索引擎优化 (SEO)

### 技术 SEO
- [ ] 每个页面有独特的 `<title>` 和 `<meta description>`
- [ ] 添加 Open Graph 标签（分享到社交媒体的预览）
- [ ] 生成 `sitemap.xml`（Jekyll 插件或手写）
- [ ] `robots.txt` 文件
- [ ] 图片加 `alt` 属性
- [ ] 语义化 HTML（`<main>`, `<nav>`, `<article>` 等）
- [ ] 提交到 Google Search Console

### 内容 SEO
- 围绕长尾关键词写文章（如 "最好的免费 JSON 格式化工具"）
- 每篇文章至少 800 字
- 内链策略：文章之间互相链接
- 标题层级合理（h1 → h2 → h3）

## 二、社交媒体推广

### Twitter/X
- 分享工具开发过程（#buildinpublic）
- 发布实用小技巧，附带网站链接
- 参与 #indiehackers #buildinpublic 话题

### Reddit
- 在 r/webdev, r/programming, r/InternetIsBeautiful 分享
- 提供价值，不要纯广告

### 中文社区
- V2EX — 在"分享创造"节点发布
- 知乎 — 写相关技术文章
- 掘金 — 分享开发过程和工具
- B站 — 录制工具使用教程

### Product Hunt
- 上线后在 Product Hunt 发布
- 准备好产品截图、描述、首条评论
- 提前联系 potential upvoters

## 三、内容营销

### 博客文章选题
1. "10个程序员必用的在线工具"
2. "我是如何用 GitHub Pages 搭建这个网站的"
3. "独立开发者如何用 AI 提高10倍效率"
4. "零成本搭建一个赚钱的网站"
5. "我的第一个 $100：独立开发者变现复盘"

### 内容分发
- 一篇写好，多平台分发
- Medium / Dev.to 同步发文（加 canonical link 回主站）

## 四、社区建设
- 创建 GitHub 仓库（工具开源，增加曝光）
- 在工具页面加"在 GitHub 上 Star"按钮
- 邮件列表：每两周发一次工具更新/新文章

## 五、付费推广（预算充足后）
- Google Ads（针对工具类关键词）
- Twitter/X Ads
- 赞助技术 Newsletter

## 推广节奏

| 周次 | 行动 |
|------|------|
| W1 | 网站上线，提交 Google Search Console |
| W2 | 发 Twitter/X #buildinpublic 第一条帖子 |
| W3 | 在 V2EX/Reddit 发布工具 |
| W4 | 写第一篇推广博客，多平台分发 |
| W6 | Product Hunt 发布 |
| W8 | 复盘数据，调整策略 |
